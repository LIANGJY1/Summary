## 2. 车载物理硬件与底层架构基础

在深入 Android Automotive 之前，必须先了解车载环境下的硬按键物理接入方式，这是决定上层架构设计的根基。

### 2.1 常见车载硬按键形态

- **方向盘按键 (SWC, Steering Wheel Controls)**：如音量加减、上一曲 / 下一曲、语音唤醒 (VR, Voice Recognition)、接听 / 挂断电话等。
- **中控面板实体按键 (Fascia / Panel Buttons)**：如空调控制 (HVAC, Heating, Ventilation and Air Conditioning)、双闪灯、Home 键、车辆设置快捷键等。
- **多功能旋钮 (Rotary Controller)**：常用于菜单导航或音量调节，支持旋转、按下及拨动。

### 2.2 硬件接入与信号传输链路

- **MCU / CAN / LIN 总线接入**：大多数车载硬按键并不直接连接到车机系统级单芯片 (SoC, System on Chip) 的通用输入输出 (GPIO, General-Purpose Input/Output)，而是先接入方向盘游丝下方的模块或车身控制模块 (BCM, Body Control Module)，通过控制器局域网 (CAN, Controller Area Network) 或局部互联网络 (LIN, Local Interconnect Network) 总线将信号发送至车机微控制器 (MCU, Microcontroller Unit)。
- **MCU 到 Android SoC 的通信**：MCU 接收到按键信号后，通常通过串行外设接口 (SPI, Serial Peripheral Interface) 或通用异步收发传输器 (UART, Universal Asynchronous Receiver-Transmitter) 等高速串口将数据打包上报至 Android / Linux 侧的 MCU 守护进程。

## 3. Android Automotive OS (AAOS) 硬按键架构

本文档主要描述 Android 13 Automotive 操作系统中，针对硬按键业务的层级软件架构。主要设计目标为：

- **解耦设计**：通过 Android 接口定义语言 (AIDL, Android Interface Definition Language) 与硬件抽象层 (HAL, Hardware Abstraction Layer) 隔离硬件变化，上层无感知。
- **统一输入管理**：系统级通用按键业务由 `CarInputService` 集中处理，便于扩展。
- **模块化服务**：非系统级通用按键业务直接对接应用，高效灵活。

### 3.1 整体架构与处理链路

用户按下硬按键后，事件的典型流转链路如下：

1. **硬件通信层**：MCU 通过 SPI 将事件分别上报至 Android 和 Linux 侧的 MCU 通信服务。
2. **硬件抽象层 (VHAL, Vehicle Hardware Abstraction Layer)**：Android 侧经 `VehicleHal` 将按键信号转换为标准属性，通过 AIDL 传递至 `CarService`。
3. **核心服务层 (CarService)**：
    - **系统级通用业务**（如音量调节、媒体控制、电话控制）：由 `CarInputService` 集中拦截并直接处理。
    - **特定业务按键**（如唤醒特定应用）：通过 `CarPropertyService` 等服务分发回调。
4. **应用层响应**：手车互联、语音助手等应用通过监听原生 `KeyEvent` 或特定车辆属性完成最终业务响应。

### 3.2 架构分层说明

| 层级 | 组件 | 职责描述 |
| :--- | :--- | :--- |
| **APP 应用层** | 语音助手、车载 vlog、组团通话、DVR、手车互联等 | 处理对应硬按键事件的具体业务逻辑。 |
| **Framework-API 层** | `CarPropertyManager`、`TelecomManager`、`MediaSessionManager` 等 | 提供稳定的 Binder 接口封装，供应用层调用，隐藏服务实现细节。 |
| **Framework-Service 层** | `CarInputService`、`CarPropertyService` | 硬按键的业务逻辑中枢。`CarInputService` 处理系统通用硬按键业务，`CarPropertyService` 上报事件给对应的 APP。 |
| **VHAL 层** | AIDL VHAL 实现 | 将 Android 标准属性映射到车辆私有总线协议，处理与硬件的直接通信。 |

### 3.3 CarInputService 事件处理相关组件说明

| 组件 | 类型 | 核心职责 |
| :--- | :--- | :--- |
| **MediaSessionManager / MediaController** | 媒体会话管理 | 支持播放 / 暂停 / 上一曲 / 下一曲等媒体按键。 |
| **TelecomManager** | 通话控制接口 | 通话状态获取和控制（响应接听或挂断）。 |
| **CarAudioService** | 音量调节接口 | 拦截音量按键，控制当前播放音源的音量。 |
| **CarInputManager** | Input 事件控制接口 | 原始 `KeyEvent` 的监听和处理分发。 |
